# VSS Package Maintainer's Guide

This guide details how to build, maintain, and publish packages for the VSS Programming Language across major operating systems and package managers.

## 📦 Directory Structure

- `.github/workflows/release.yml` - GitHub Actions CI/CD release workflow
- `packaging/homebrew/vss.rb` - Homebrew Formula for macOS/Linux
- `packaging/chocolatey/` - Chocolatey package definitions and PowerShell installers
- `packaging/winget/vss.yaml` - Winget manifest for Windows Package Manager
- `packaging/debian/` - Debian/Ubuntu `.deb` metadata and rules
- `packaging/rpm/vss.spec` - RPM specification for Fedora/RHEL/CentOS
- `packaging/arch/PKGBUILD` - Arch Linux build definitions

---

## 🚀 Releasing a New Version

To release a new version of VSS:

1. **Update Code References**:
   Update version string definitions in:
   - `vss/src/cli.c` (inside `run_cli` version info)
   - `packaging/homebrew/vss.rb` (`url` and version checks)
   - `packaging/chocolatey/vss.nuspec` (`<version>`)
   - `packaging/winget/vss.yaml` (`PackageVersion` and `InstallerUrl`)
   - `packaging/debian/control` (`Version`)
   - `packaging/rpm/vss.spec` (`Version`)
   - `packaging/arch/PKGBUILD` (`pkgver`)

2. **Commit and Tag**:
   ```bash
   git add .
   git commit -m "release: bump version to v1.0.0"
   git tag -a v1.0.0 -m "VSS v1.0.0 Release"
   git push origin main --tags
   ```

3. **CI/CD Execution**:
   - The GitHub Actions workflow will trigger on the pushed tag.
   - It will compile native binaries for Linux, macOS, and Windows.
   - It will package them into `.tar.gz`, `.deb`, and `.zip` archives.
   - It will create a GitHub Release and upload the assets.

4. **Update SHA-256 Hashes**:
   Download the released assets from GitHub, calculate their SHA-256 hashes, and update:
   - `sha256` in `packaging/homebrew/vss.rb`
   - `checksum64` in `packaging/chocolatey/tools/chocolateyinstall.ps1`
   - `InstallerSha256` in `packaging/winget/vss.yaml`
   - `sha256sums` in `packaging/arch/PKGBUILD`

## 🌐 Publishing to Package Managers

### 1. Winget (Windows)
Submit the manifest to [winget-pkgs](https://github.com/microsoft/winget-pkgs):
```bash
# Install wingetcreate tool
winget install wingetcreate

# Submit updated manifest
wingetcreate submit VSS.VSS --version 1.0.0 --urls https://github.com/siddharth-1118/vss-language/releases/download/v1.0.0/vss-windows-amd64.zip
```

### 2. Homebrew (macOS)
Submit to [homebrew-core](https://github.com/Homebrew/homebrew-core) or maintain an official Tap (`homebrew-vss`):
- Fork `homebrew-core`, copy `packaging/homebrew/vss.rb` into `Formula/v/vss.rb`, and submit a Pull Request.

### 3. Chocolatey (Windows)
Build and push the `.nupkg`:
```bash
cd packaging/chocolatey
choco pack
choco push vss.1.0.0.nupkg --api-key <YOUR_API_KEY>
```

### 4. Debian / Ubuntu (`apt`)
Host a PPA (Personal Package Archive) on Launchpad:
- Upload the debian source package generated by `debuild -S` to Launchpad, which will build and distribute it to Ubuntu users.

### 5. Fedora / Red Hat (`dnf`)
Host a repository on [Copr](https://copr.fedorainfracloud.org/):
- Create a new project, upload the RPM `.spec` file, and point it to the GitHub Release tarball.

### 6. Arch User Repository (AUR)
Create a Git repository on `aur.archlinux.org` for `vss-bin` or `vss`:
- Push the `PKGBUILD` and `.SRCINFO` to the repository.
